# Pre-PR Self-Review Checklist

Before opening a pull request, I confirm that:

- [ ] The code does exactly what the PR title and description claim
- [ ] All tests pass locally with no failures or errors
- [ ] I have added or updated tests if behavior changed
- [ ] The README and documentation are accurate and up to date
- [ ] There are no debug statements (print, breakpoint, console logs)
- [ ] The PR focuses on one clear, single responsibility
- [ ] I reviewed my own code for readability and clarity
- [ ] No unnecessary files or commented-out code are included
